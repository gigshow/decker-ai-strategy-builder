# 사용법 가이드

**한 페이지 요약.** 신규 사용자가 안내 없이 첫 시그널 조회 가능. *말만 하면* 시그널·포지션·주문.

---

## 채팅 vs 대시보드

| 구분 | 채팅 (Telegram 권장, OpenClaw 스킬) | 대시보드 (decker-ai.com) |
|------|------------------------|--------------------------|
| **시그널·시세** | ✅ | ✅ |
| **상의·전략** | ✅ (이 시그널 어떻게 할까?) | ✅ |
| **포지션·주문·청산** | ✅ | ✅ |
| **자동주문·뉴스** | ✅ | ✅ |
| **폴리마켓·HL** | ✅ | ✅ |
| **연동 코드 발급** | ❌ | ✅ (decker-link-telegram) |
| **API 키·실행 모드** | ❌ | ✅ |
| **전략 빌더·포트폴리오** | ❌ | ✅ |

→ **연동은 반드시 대시보드에서.** 채팅으로 "말만 하면" 시그널·주문 가능.

## 주요 명령어 11개

| # | 명령어 | 설명 |
|---|--------|------|
| 1 | 비트코인 시그널 알려줘 | 시그널·진행도(progress_pct)·전략 |
| 2 | 추천종목 알려줘 | 현재 추천 종목 목록 |
| 3 | 비트코인 얼마 / ETH 시세 | 실시간 시세 |
| 4 | 포지션 보여줘 / 수익현황 | 보유 포지션·손익 |
| 5 | ETH 0.01 매수해줘 | 수동 주문 (승인 후 실행) |
| 6 | ETH 청산해줘 | 포지션 청산 |
| 7 | 이더리움 자동주문 해줘 | 자동주문 규칙 설정 |
| 8 | 이 시그널 어떻게 할까? | 상의·전략 (오퍼레이션 룰북) |
| 9 | 뉴스 보여줘 | 마켓 뉴스 |
| 10 | 시장 상태 / 폴리마켓 | 바이어스·펀딩·예측시장 |
| 11 | 하이 / 단계별로 알려줘 | 서비스 리스트·5단계 체험 가이드 |

## 3분 온보딩

1. **가입** — decker-ai.com 회원가입
2. **연동** — decker-link-telegram에서 코드 발급 → Telegram @deckerclawbot에서 `/start {코드}`
3. **바로 사용** — "비트코인 시그널 알려줘"

---

## 5단계 체험 가이드 (처음인데 / 단계별로 알려줘)

| 단계 | 체험 | 사용자 액션 | 성공 기준 |
|------|------|-------------|-----------|
| **1** | 모의거래 | 가입→연동→"ETH 0.01 매수해줘"→승인 | 주문 실행, 포지션 생성 |
| **2** | 자동주문·청산 | "이더리움 자동주문 해줘"→"ETH 청산해줘" | 규칙 설정, 포지션 청산 |
| **3** | 실거래 | 설정에서 Binance API 키 등록→"ETH 매수해줘" | Binance 주문 |
| **4** | Hyperliquid | 설정에서 HL 키, exchange_preference=HL→"HL BTC 매수해줘" | HL 주문 |
| **5** | Polymarket | 설정에서 PM 연동→"Polymarket 시장 yes 10 매수" | 예측시장 주문 |

---

## HL·Polymarket 실주문 (설정 필요)

| 거래소 | 설정 경로 | 예시 명령 |
|--------|-----------|-----------|
| **Hyperliquid** | decker-ai.com → 설정 → 거래소 API → Hyperliquid | HL BTC 0.01 매수해줘 |
| **Polymarket** | decker-ai.com → 설정 → 거래소 API → Polymarket | Polymarket 시장 yes 10 매수 |

**DEX 에이전트 전용 지갑 (HL/PM 필수)**: 메인 지갑 개인키는 절대 입력하지 말 것. 새 지갑 생성 → 거래용 자금만 입금.
