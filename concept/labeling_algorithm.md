# 시계열 라벨링 알고리즘

---

## 역할

시계열 데이터를 **오브젝트(대상)**로 평가하여 라벨을 부착합니다. 시그널 발생 조건을 정의합니다.

이 알고리즘이 DECKER 시그널 엔진의 **원천 데이터**를 생성하는 핵심 IP입니다.

---

## 핵심 원리: 객체 기반 평가

일반 트레이딩 봇은 인디케이터(RSI, MACD 등)로 신호를 생성합니다.

DECKER의 라벨링 알고리즘은 **시장 구조(Object)**에 집중합니다. 직전 고점/저점 같은 객체에 도달했을 때 **프라임(1/2)** 평가를 하여 추세 전환 여부를 판단합니다.

이 **객체(Object) 평가 + 상태(State) 계산** 방식은 일반 인디케이터 기법과 본질적으로 다릅니다.

---

## 라벨 구성

| 라벨 | 의미 |
|------|------|
| **1/2 대상 + 1/2 프라임** | 평가 대상과 평가 기준(프라임)의 조합 — 대상에 대한 평가 |
| **S** | 시그널 (Signal) — 진입 조건 충족 |
| **T** | 터치 (Touch) — 향후 시그널 진입점. 중첩의 의미 포함 |
| **1** | 기타 상태 |

S/T 라벨은 중첩될 수 있습니다. T는 향후 시그널 진입의 기준점이 되며, 동일 시점에 복수의 스윙에서 T가 발생하면 **중첩 시그널**로 강도가 높아집니다.

---

## 프라임 평가 (Prime)

객체 도달 시점에서 프라임 단계를 할당합니다.

| 단계 | 의미 |
|------|------|
| **1/2** | 평가 대상의 절반 지점 도달 — 1차 평가 |
| **1/2'** (2프라임) | 평가봉 돌파 — 시그널 발생 트리거 |

예: 가격이 이전 고점의 1/2 지점에 도달하면 `1/2`로 평가하고, 해당 평가봉(2프라임)을 현재가가 브레이크하면 시그널이 발생합니다.

---

## 오브젝트와 스윙

오브젝트는 시점에 따라 **멀티스윙**으로 구성됩니다.

| 스윙 | 의미 |
|------|------|
| **A** | 메인스윙 — 트렌드의 키 방향. 평가 대상 2개 이상이 현재가 기준 위/아래에 위치할 때, 가격 상승/하락으로 평가 |
| **B** | 서브스윙 — 메인스윙의 반대 방향 조정 |
| **C** | 연결스윙 — 시장 청산·진입·리버스가 일어나는 구간 |

### 스윙 계산

스윙 조합으로 시장 구조를 계산합니다:

| 조합 | 구성 |
|------|------|
| **AB** | 메인스윙 + 서브스윙 (트렌드 내 조정) |
| **AC** | 메인스윙 + 연결스윙 (트렌드 전환 구간) |
| **BB** | 서브스윙 연속 (조정 지속) |
| **BC** | 서브스윙 + 연결스윙 (조정 후 전환) |

각 스윙의 **평가봉(2프라임)**을 현재가가 브레이크할 때 **시그널 발생**.

---

## 시그널 발생 조건

```
1. 시계열 데이터 수집
2. 오브젝트(고점/저점 등) 식별
3. 프라임 평가 (1/2, 1/2')
4. 스윙 분류 (A/B/C)
5. 스윙 조합 계산 (AB, AC, BB, BC)
6. 평가봉(2프라임) 브레이크 감지
7. → 시그널(S) 발생
```

---

## 파이프라인 내 위치

```
시계열 데이터 → [라벨링 알고리즘] → 라벨된 시그널 (S, T, 1)
    → State Engine (progress_pct, status)
    → 오퍼레이션 룰북
    → 전략 반환
```

라벨링 알고리즘은 파이프라인의 **첫 번째 단계**이며, 이후 모든 계산의 입력이 됩니다.
